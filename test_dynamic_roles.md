# 动态角色与权限配置功能测试指南

## 功能概述
已成功实现"动态角色与权限配置"功能，该功能允许管理员在【人事与权限管理】页面中动态管理角色和权限。

## 已实现的功能

### 1. 数据库架构扩展
- ✅ 创建了三个新表：`dynamic_roles`、`permissions`、`role_permissions`
- ✅ 修改了`users`表，将`role`字段改为`role_id`外键
- ✅ 创建了数据库迁移文件
- ✅ 创建了数据初始化脚本

### 2. 后端API接口
- ✅ 动态角色管理API (`/dynamic-roles/roles`)
  - 获取所有角色
  - 创建新角色
  - 更新角色信息
  - 删除角色（软删除）
  - 更新角色权限
- ✅ 权限管理API (`/dynamic-roles/permissions`)
  - 获取所有权限
  - 创建新权限
  - 更新权限信息
  - 删除权限（软删除）
- ✅ 角色权限关联API
  - 更新角色的权限配置
  - 获取角色的所有权限
- ✅ 目录API
  - 获取角色目录（用于下拉选择）
  - 获取权限目录（按分类分组）

### 3. 前端界面
- ✅ 创建了`DynamicRolesConfig.vue`组件
  - 左侧：角色列表（支持搜索、筛选）
  - 右侧：权限配置（按分类分组）
  - 角色管理：新增、编辑、启用/禁用、删除
  - 权限管理：新增、编辑、启用/禁用
  - 权限配置：全选、分类选择、单个权限选择
- ✅ 集成到现有人事与权限管理页面
  - 在【人事与权限管理】卡片右上角添加"动态角色配置"按钮
  - 点击按钮打开动态角色配置模态框

### 4. API客户端
- ✅ 创建了`dynamicRoles.ts` API客户端
- ✅ 定义了完整的TypeScript类型

## 使用说明

### 1. 启用功能
1. 启动MySQL数据库服务器
2. 运行数据库迁移：
   ```bash
   cd server
   alembic upgrade head
   ```
3. 初始化数据：
   ```bash
   cd server
   python db/init_dynamic_roles.py
   ```
4. 启动后端服务器
5. 启动前端开发服务器

### 2. 访问功能
1. 登录系统（需要管理员权限）
2. 导航到【人事与权限管理】页面
3. 点击右上角的"动态角色配置"按钮
4. 在弹出的模态框中管理角色和权限

### 3. 功能操作指南

#### 角色管理
- **新增角色**：点击"新增角色"按钮，填写角色信息
- **编辑角色**：点击角色项右侧的菜单按钮，选择"编辑"
- **启用/禁用角色**：点击角色项右侧的菜单按钮，选择"启用"或"禁用"
- **删除角色**：点击角色项右侧的菜单按钮，选择"删除"（仅限无用户分配的角色）

#### 权限管理
- **新增权限**：点击"新增权限"按钮，填写权限信息
- **编辑权限**：点击权限项右侧的菜单按钮，选择"编辑"
- **启用/禁用权限**：点击权限项右侧的菜单按钮，选择"启用"或"禁用"

#### 权限配置
1. 在左侧角色列表中选择一个角色
2. 在右侧权限配置区域配置权限：
   - 使用"全选"按钮选择所有权限
   - 使用"清空"按钮清除所有权限
   - 按分类选择权限
   - 单个权限选择
3. 点击"保存权限"按钮保存配置
