# 项目重构规划技能

## 概述
此技能用于规划和管理项目重构工作，特别是当用户希望使用新技术栈重写现有项目时。技能提供系统化的规划方法，确保重构工作有序进行。

## 使用场景
- 用户希望用新技术栈重写现有项目
- 项目需要重大架构变更
- 需要从零开始规划新项目结构
- 需要分解复杂重构任务为可管理的小任务

## 核心功能

### 1. 项目分析
- 分析现有项目结构和功能
- 识别技术债务和重构重点
- 评估现有业务逻辑和界面设计

### 2. 架构设计
- 设计新的技术架构
- 规划模块划分和依赖关系
- 设计数据迁移策略

### 3. 任务分解
- 将重构工作分解为具体任务
- 估算工作量和优先级
- 创建详细的任务清单

### 4. 进度跟踪
- 跟踪重构进度
- 管理任务状态
- 提供进度报告

## 工作流程

### 阶段1：需求分析
1. 理解用户的技术栈要求
2. 分析现有项目功能
3. 确定重构范围和目标

### 阶段2：架构设计
1. 设计新的技术架构
2. 规划数据库迁移
3. 设计API接口

### 阶段3：任务规划
1. 创建详细的任务清单
2. 设置优先级和时间线
3. 分配资源需求

### 阶段4：实施跟踪
1. 监控实施进度
2. 调整计划应对变化
3. 确保质量符合要求

## 输出物
- 项目重构规划文档
- 技术架构设计图
- 详细任务清单
- 进度跟踪报告

## 最佳实践
1. **渐进式重构**：优先重构核心功能，逐步扩展
2. **保持兼容**：确保新老系统可以并行运行
3. **测试驱动**：为每个功能编写测试用例
4. **文档完整**：保持架构和API文档更新
5. **团队协作**：确保所有相关人员了解重构计划

## 工具建议
- 使用TodoWrite工具管理任务
- 使用图表工具绘制架构图
- 使用版本控制管理代码变更
- 使用项目管理工具跟踪进度

## 成功指标
- 所有功能按计划实现
- 新系统性能优于旧系统
- 代码质量显著提升
- 团队对新技术栈掌握良好
- 用户满意度提高
